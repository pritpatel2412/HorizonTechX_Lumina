Build a full-stack social media web application called "LUMINA" — a premium social network. This is a complete, production-ready product with a next-level UI.

---

## TECH STACK
- Backend: Node.js + Express.js
- Database: SQLite with better-sqlite3 (file-based, no setup needed)
- Frontend: HTML + CSS + Vanilla JavaScript (served via Express)
- Authentication: JWT (jsonwebtoken) + bcryptjs
- File Uploads: multer (for profile pictures and media)
- No external CSS frameworks — all CSS written from scratch

## DATABASE SCHEMA

Create these SQLite tables:

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT UNIQUE NOT NULL,
  display_name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  bio TEXT DEFAULT '',
  avatar_url TEXT DEFAULT '/default-avatar.png',
  cover_url TEXT DEFAULT '',
  website TEXT DEFAULT '',
  location TEXT DEFAULT '',
  verified INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  content TEXT NOT NULL,
  image_url TEXT DEFAULT '',
  post_type TEXT DEFAULT 'post', -- 'post', 'story', 'moment'
  expires_at DATETIME DEFAULT NULL, -- for stories (24hr)
  views INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  content TEXT NOT NULL,
  parent_comment_id INTEGER DEFAULT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE likes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  post_id INTEGER,
  comment_id INTEGER,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, post_id),
  UNIQUE(user_id, comment_id)
);

CREATE TABLE follows (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  follower_id INTEGER NOT NULL,
  following_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(follower_id, following_id),
  FOREIGN KEY (follower_id) REFERENCES users(id),
  FOREIGN KEY (following_id) REFERENCES users(id)
);

CREATE TABLE stories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  media_url TEXT NOT NULL,
  media_type TEXT DEFAULT 'image',
  caption TEXT DEFAULT '',
  bg_color TEXT DEFAULT '#7c6aff',
  text_content TEXT DEFAULT '',
  views INTEGER DEFAULT 0,
  expires_at DATETIME DEFAULT (datetime('now', '+24 hours')),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE story_views (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  story_id INTEGER NOT NULL,
  viewer_id INTEGER NOT NULL,
  viewed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(story_id, viewer_id)
);

CREATE TABLE notifications (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  recipient_id INTEGER NOT NULL,
  sender_id INTEGER NOT NULL,
  type TEXT NOT NULL, -- 'like', 'comment', 'follow', 'mention'
  post_id INTEGER DEFAULT NULL,
  comment_id INTEGER DEFAULT NULL,
  read INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (recipient_id) REFERENCES users(id),
  FOREIGN KEY (sender_id) REFERENCES users(id)
);

CREATE TABLE saved_posts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  post_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, post_id)
);
```

## BACKEND — Express.js API

### server.js
- Express app on port 3000
- Middleware: cors, express.json, express.urlencoded, cookie-parser
- Static files from /public
- JWT middleware: reads token from Authorization header OR cookie named 'lumina_token'
- Routes: /api/auth, /api/posts, /api/users, /api/stories, /api/notifications
- Serve HTML pages: GET / → index.html, GET /feed → feed.html, GET /profile/:username → profile.html, GET /explore → explore.html
- Auto-delete expired stories (run cleanup every hour with setInterval)

### Auth Routes (/api/auth)
- POST /register: validate username(3-20 chars, alphanumeric+underscore), email, password(min 8 chars); hash password with bcrypt(10); create user; return JWT + user object
- POST /login: find user by email or username; compare password; return JWT (expires in 7d) + user object
- GET /me: protected, return current user data
- PUT /profile: protected, update display_name, bio, website, location, avatar (multer)
- POST /change-password: protected, verify old password, update new

### Posts Routes (/api/posts)
- GET /feed: protected, paginated (limit 10, offset param), get posts from followed users + own posts, ordered by created_at DESC, include: author info, like count, comment count, whether current user liked/saved, first 2 comments
- GET /explore: public, trending posts (most likes in last 7 days), paginated
- GET /:id: single post with full comments tree
- POST /: protected, create post (content required, image optional via multer, post_type optional)
- DELETE /:id: protected, only post owner can delete
- POST /:id/like: protected, toggle like, return {liked: bool, count: int}
- POST /:id/save: protected, toggle save, return {saved: bool}
- GET /:id/comments: get comments for post
- POST /:id/comments: protected, add comment, create notification
- DELETE /comments/:commentId: protected, only comment owner can delete
- POST /comments/:commentId/like: toggle like on comment

### Users Routes (/api/users)
- GET /search?q=: search users by username or display_name
- GET /:username: get user profile + stats (post count, followers, following)
- GET /:username/posts: get user's posts paginated
- GET /:username/followers: list followers
- GET /:username/following: list following
- POST /:username/follow: protected, toggle follow, return {following: bool, count: int}, create notification
- GET /suggestions: protected, suggest users to follow (not already following, most followers)

### Stories Routes (/api/stories)
- GET /: protected, get active stories (not expired) from followed users + own, grouped by user
- POST /: protected, upload story (image via multer, or text-only with bg_color and text_content)
- POST /:id/view: protected, record view, return view count
- GET /:id/viewers: protected (only story owner), list who viewed
- DELETE /:id: protected, story owner only

### Notifications Routes (/api/notifications)
- GET /: protected, get user's notifications paginated, include sender info
- POST /read/:id: mark one as read
- POST /read-all: mark all as read
- GET /unread-count: return {count: int}

## SECURITY & POLISH

- All API routes that modify data require valid JWT
- SQL injection prevented by using parameterized queries (never string concat SQL)
- Passwords never returned in any API response
- File upload: validate mime type (images only), max 5MB, rename to UUID
- 404 page: stylized with the design system
- All forms show loading spinner on submit button while request is in flight
- Empty states: when feed is empty, show a beautiful illustration-style message "Nothing here yet — follow some people!"
- Error boundaries: if any section fails to load, show retry button
